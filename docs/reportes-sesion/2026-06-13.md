## Qué se hizo

### 1. Gemini 2.5 Flash — modelo y timeout
- `apps/content/services/ai_generation_service.py`: cambio de `gemini-1.5-flash` (deprecado junio 2026) → `gemini-2.5-flash`.
- Timeout HTTP subido de 30 s → 90 s (generación de 20 preguntas tardaba ~60 s).

### 2. Publicar preguntas directo al generar
- `_save_questions()`: `status="borrador"` → `status="publicada"` (opción temporal hasta que exista el estudio de banco).

### 3. Admin: banco de 20 preguntas por nivel
- `generar_preguntas_ia_action`: acepta `levels` (lista de checkboxes) en lugar de un solo `level`.
- Template `generate_ai_questions_confirm.html`: 3 checkboxes (niveles 1/2/3, todos marcados), count default 20, max 30.
- Genera N preguntas × cada nivel seleccionado en un solo clic.

### 4. Fix CSS: radio buttons tomaban ancho 100%
- `.quiz-choice__input`: dimensiones fijas (`width/height: 1.1rem`, `padding: 0`) para neutralizar la regla global `input { width: 100% }`.
- `.quiz-choice`, `.quiz-choice__text`: fixes de layout para que el texto no se corte.
- CSS version: v24 → v27.

### 5. Fix crítico: "Enviar respuestas" no hacía nada
- **Causa raíz:** `<select required>` en el panel "Reportar error" (dentro del mismo `<form>` del quiz) activaba la validación HTML5 silenciosamente.
- **Fix:** `required` removido del select (validación server-side suficiente).
- **Fix secundario:** botón "Enviar reporte" era `type="submit"` → cambiado a `type="button"` con su propio `hx-post` para no triggear el form padre.

### 6. Fix CSP: onclick inline bloqueado
- Botones Cancel y "Reportar error" usaban `onclick="..."` → bloqueados por CSP (`script-src 'self' 'nonce-{nonce}'`).
- Reemplazados por atributos `data-quiz-cancel` y `data-quiz-report-toggle`.
- Handler delegado en `<script nonce="{{ csp_nonce }}">` en `base.html` con `document.addEventListener('click', ...)`.
- **Bug fix adicional:** `document.body.addEventListener` → `document.addEventListener` (body es null en `<head>`).

### 7. HTMX: scroll anchoring al enviar/cancelar
- `hx-swap` en form y botones de resultados: `show:#quiz-level-{{ level }}:top` para que la página no salte al cambiar el contenido.
- `scroll-margin-top: 90px` en `.quiz-level` y `.quiz-section` (compensa el sticky header de 77 px).

### 8. Legibilidad de explicaciones paso a paso
- Nuevo templatetag `format_steps` en `apps/core/templatetags/markdown_tags.py`.
- Regex que detecta `\d. ` y añade `<br>` entre pasos; escapa HTML, preserva decimales.
- Prompt de generación IA actualizado para pedir pasos numerados separados por `\n`.

### 9. UX post-evaluación: botones contextuales
- `quiz_results.html` reescrito con footer contextual:
  - **Práctica:** "Ir a Evaluación" (si no aprobó y no alcanzó max), "Practicar de nuevo", "Volver a niveles".
  - **Evaluación:** "Volver a intentar" (si quedan intentos), "Ir a Práctica", "Volver a niveles".

### 10. HTMX error responses visibles
- `htmx:beforeSwap` en `base.html`: `shouldSwap=true` para respuestas 4xx (antes se silenciaban).

### 11. Handoff: Estudio de banco de preguntas
