## Decisiones importantes
- **Normalización de defaults en creación inline:** En lugar de lanzar errores de validación si campos internos como `order`, `is_active` o `is_published` no se proveen desde los modales sencillos, el endpoint `publish_inline_create` los normaliza asignando defaults válidos (`0`, `true`, `true`), evitando sobrecargar la UI de creación inline.
- **Endpoint independiente para módulos:** Aunque el módulo es opcional en Fase 1, se creó el endpoint `module_options` para filtrar los módulos de forma reactiva al cambiar de asignatura, igualando el comportamiento dinámico de los temas.
